# ADR-0001: 모노레포 구조 채택

- 상태: 승인
- 날짜: 2026-03-15

## 배경
프론트엔드, 백엔드, 공통 계약, 문서, 프롬프트를 하나의 작업 컨텍스트 안에서 관리할 필요가 있다.

## 결정
다음 구조의 모노레포를 채택한다.

- `apps/web`
- `apps/api`
- `packages/contracts`
- `docs/*`

## 이유
- Codex가 한 저장소 안에서 구조를 읽고 반영하기 쉽다.
- 공통 타입과 문서를 한 곳에서 유지하기 쉽다.
- 페이지별 프롬프트와 실제 API 계약을 같은 저장소 안에서 추적하기 좋다.
