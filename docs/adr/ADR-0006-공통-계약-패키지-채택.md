# ADR-0006: 공통 계약 패키지 채택

- 상태: 승인
- 날짜: 2026-03-26

## 배경

`apps/web`는 화면 loader 안에 raw API 응답 타입을 따로 들고 있었고, `apps/api`는 `apps/api/prompts/*/output.schema.json`과 Pydantic schema를 별도로 유지하고 있었다. 이 구조는 다음 문제를 만들었다.

- web 타입과 API schema drift 위험
- prompt schema 수정 시 web loader 타입 불일치 위험
- 공통 메타 구조(`asOf`, `sourceRefs`, `missingData`, `confidence`)의 기준 위치 부재

## 결정

공통 계약 패키지 `packages/contracts`를 추가한다.

- runtime JSON schema 단일 소스는 `packages/contracts/schemas/*.schema.json`
- web 타입 단일 소스는 `packages/contracts/src/index.ts`
- API prompt loader는 `packages/contracts/schemas/*.schema.json`을 우선 읽는다
- `apps/api/prompts/*/system.md`는 계속 페이지별 프롬프트 문자열만 담당한다
- Python Pydantic schema는 별도 구현을 유지하되, 같은 계약을 따르도록 검증과 문서로 관리한다

## 이유

- Python과 TypeScript를 한 런타임 모델로 억지 공유하지 않고도 계약 단일 소스를 둘 수 있다
- LLM structured output, provider validation, web loader 타입을 같은 계약 축에 맞출 수 있다
- 기존 구현을 갈아엎지 않고 최소 변경으로 drift를 줄일 수 있다

## 영향

- `packages/contracts`가 모노레포 필수 폴더가 된다
- web 타입 shim은 `apps/web/lib/research/types.ts`를 통해 유지한다
- API runtime schema 위치는 `apps/api/prompts/*/output.schema.json`에서 `packages/contracts/schemas/*.schema.json`로 이동한다
