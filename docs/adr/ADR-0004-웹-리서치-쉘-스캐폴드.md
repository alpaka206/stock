# ADR-0004: 웹 리서치 셸과 feature-first 구조

- 상태: 승인
- 날짜: 2026-03-16
- 갱신: 2026-05-16

## 배경

Stock Desk는 `/overview`, `/radar`, `/stocks/[symbol]`, `/history`를 중심으로 같은 리서치 흐름을 반복한다. 공통 셸과 화면별 feature 경계를 명확히 잡지 않으면 페이지마다 데이터 로드, 필터, 카드 스타일, 문구가 흩어진다.

## 결정

- `app/*/page.tsx`는 얇은 route wrapper로 유지한다.
- 공통 shell은 sidebar, topbar, theme toggle, language select를 제공한다.
- 화면 조합은 `features/*`에 둔다.
- 공통 UI는 `components/*`에 둔다.
- 개발용 fixture는 `dev/fixtures/*`에 둔다.
- 레이더 기본 화면은 경량 네이티브 테이블을 사용한다.
- 차트는 `ResearchLineChart`를 공통으로 확장한다.
- 판단 기록은 서버 API로 저장한다.

## 이유

- Server Component에서 데이터 로드를 먼저 처리할 수 있다.
- Client Component 범위를 검색, 필터, 차트 선택, 저장 같은 상호작용으로 줄일 수 있다.
- 대형 그리드 라이브러리를 기본 경로에서 제거해 초기 JS와 렌더 비용을 줄인다.
- fixture와 실제 API의 데이터 출처 표시 규칙을 공통화할 수 있다.

## 영향

- `apps/web/app`, `apps/web/components`, `apps/web/features`, `apps/web/dev/fixtures` 구조가 기준이 된다.
- 레이더의 표 기능은 필요한 범위만 직접 구현한다.
- 더 큰 표 기능이 필요해지면 TanStack Table 같은 headless 옵션을 먼저 검토한다.
