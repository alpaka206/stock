# 목표
`stock`를 국장과 미장을 함께 다루는 완성형 주식 분석 사이트로 발전시킨다.

# 반드시 읽을 문서
- docs/design/stock-reference-blueprint.md
- docs/design/chart-pattern-atlas.md
- docs/design/design-memory.md
- docs/architecture/page-manifest.yaml
- docs/architecture/component-manifest.yaml

# 제품 기준
- 초보자도 쉽게 읽을 수 있어야 한다.
- 숙련 사용자도 "이 사이트 보기 좋네"라고 느낄 정도로 차트, 수급, 공시, 뉴스, 캘린더 문맥이 잘 정리되어야 한다.
- 국장과 미장을 분리된 서비스처럼 만들지 말고, 하나의 연구 흐름 안에서 자연스럽게 비교 가능해야 한다.

# 핵심 UX 요구사항
- overview에서 오늘 시장 방향을 빠르게 파악한다.
- radar에서 종목을 스크리닝하고 섹터 흐름을 읽는다.
- stock detail에서 차트, 공시, 뉴스, 이벤트, 비교 종목, 해석 패널이 한 번에 이어진다.
- history에서 특정 가격 흐름과 이벤트 타임라인을 함께 본다.

# 차트 요구사항
- 기본 제공 이동평균선: 5일, 10일, 20일, 60일, 120일
- 거래량 기본 제공
- 차트 위 또는 옆에서 현재 구조를 설명형으로 요약
- 패턴 도감 또는 패턴 카드 영역을 함께 제공
- 현재 차트와 가장 유사한 패턴을 "지금 이런 차트로 진행 중이에요" 수준으로 쉽게 보여준다
- 설명 항목 예시:
  - 상승 정배열 여부
  - 하락 역배열 여부
  - 단기선 교차
  - 지지선/저항선 근접
  - 거래량 급증/급감
  - 갭 발생
  - 이벤트 이후 추세 유지 여부
  - cup with handle, double bottom, flat base 같은 패턴과의 유사도
  - 패턴 진행 단계와 무효화 조건

# 해석 문구 원칙
- 단정적 예언 금지
- 좋은 문구 예시:
  - "단기 추세는 살아 있지만 거래량이 줄어 추격은 주의가 필요합니다."
  - "20일선 위에서 안착하는지 확인이 필요한 구간입니다."
  - "최근 상승 대비 거래량이 동반되지 않아 힘이 약해질 수 있습니다."
  - "현재 흐름은 double bottom과 유사하지만 넥라인 돌파 전이라 확인 구간으로 보는 편이 안전합니다."
  - "flat base와 비슷한 횡보 구조가 진행 중이며, 박스 상단 돌파 여부를 먼저 봐야 합니다."
- 피해야 할 문구:
  - "무조건 오른다"
  - "지금 사면 된다"
  - "반드시 떨어진다"

# 데이터 요구사항
- 한국:
  - KRX 시세/시장 데이터
  - KIND 공시/상장 정보
  - DART / OpenDART 공시
- 미국:
  - SEC EDGAR 공시
  - Nasdaq 계열 시세/시장 정보
- 각 데이터 블록에 출처와 지연 여부를 표시한다.

# 디자인 요구사항
- 실제 참고 서비스의 장점은 흡수하되 복제처럼 보이면 안 된다.
- TradingView식 차트 집중력, Koyfin식 대시보드 밀도, Finviz식 스캔 속도, Nasdaq식 데이터 투명성을 조합한다.
- 초보자에게는 친절한 설명, 숙련자에게는 높은 정보 밀도를 동시에 준다.
- 카드만 많은 AI 슬롭 화면을 피하고, 시선 흐름이 분명한 연구형 레이아웃을 유지한다.
- 사용자가 준 패턴 이미지 레퍼런스처럼 도식화된 패턴 카드와 실제 종목 차트를 연결하는 UX를 적극 활용한다.

# 완료 조건
- 차트와 해석 패널이 stock detail에서 유기적으로 동작한다.
- 패턴 도감과 현재 유사 패턴 카드가 실제 종목 차트와 연결된다.
- overview / radar / history / stock detail이 각각 따로 노는 페이지가 아니라 연구 흐름으로 연결된다.
- 국장과 미장 모두 필요한 정보가 빠지지 않는다.
- 설명은 한국어로 자연스럽고, 데이터 출처는 명확하며, unavailable 이유가 드러난다.
