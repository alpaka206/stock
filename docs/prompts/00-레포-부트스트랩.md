# 목표
새 모노레포 저장소를 초기화하고, 4개 메인 페이지 구조와 문서 구조를 반영한 기본 골격을 만든다.

# 참고 문서
- AGENTS.md
- docs/architecture/page-manifest.yaml
- docs/architecture/component-manifest.yaml
- docs/design/design-memory.md

# 할 일
- `apps/web`, `apps/api`, `docs`, `packages`, `scripts` 구조를 만든다.
- 루트 `package.json`, `pnpm-workspace.yaml`, `.gitignore`, `.editorconfig`, `README.md` 기본 골격을 만든다.
- `apps/web`는 Next.js App Router 기반, `apps/api`는 FastAPI 기반으로 잡는다.
- 루트/웹/API의 AGENTS.md가 없으면 생성하고, 있으면 현재 구조에 맞게 보강한다.
- 4개 메인 페이지 route 파일을 빈 shell 형태로 만든다.
- 문서 폴더와 append-only changelog 구조를 만든다.

# 제약
- 구조를 임의로 바꾸지 않는다.
- 메인 route는 아래 4개로 고정한다.
  - `/overview`
  - `/radar`
  - `/stocks/[symbol]`
  - `/history`
- 사용자 노출 문구는 한국어로 작성한다.
- AG Grid 중심 구조를 염두에 둔다.

# 완료 조건
- `pnpm install` 후 웹앱이 부팅 가능하다.
- 4개 route 이동이 된다.
- README 실행 방법이 실제 구조와 맞는다.
- changelog 첫 항목이 추가된다.
