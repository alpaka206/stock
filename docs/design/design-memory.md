# 디자인 메모

## 방향

- 금융 서비스처럼 신뢰감이 먼저 느껴지는 화면을 만든다.
- 첫 화면은 마케팅 설명이 아니라 바로 쓰는 리서치 워크스페이스다.
- 정보는 많이 보여주되, 같은 뜻의 설명을 반복하지 않는다.
- 초보자는 시장 흐름, 종목 이유, 다음 확인 항목을 순서대로 읽을 수 있어야 한다.

## 참고한 톤

- Binance: 거래 화면의 긴장감, 노란 포인트, 어두운 수치 화면.
- Coinbase: 신뢰 중심 파란 금융 UI, 넓은 여백, 기관형 톤.
- Vercel: 흑백 기반 정밀도, 얇은 border, Geist 계열 타이포그래피.

## 토큰 원칙

- 기본 표면은 흰색과 옅은 회색을 사용한다.
- 핵심 액션은 파란색, 주의/시장 이벤트는 앰버, 상승/하락은 초록/빨강으로 분리한다.
- radius는 8px 이하를 기본으로 한다.
- 숫자는 tabular nums를 사용한다.
- letter spacing은 0을 기본으로 한다.
- 차트와 테이블은 고정 높이와 안정적인 grid를 사용해 reflow를 줄인다.

## 화면 원칙

- `/overview`: 오늘 볼 시장 흐름과 위험만 먼저 보여준다.
- `/radar`: 관심종목, 조건 감지, 섹터 흐름을 한 화면에서 훑는다.
- `/stocks/[symbol]`: 차트, 기술 지표, 뉴스, 공시, 판단 기록을 종목 중심으로 묶는다.
- `/history`: 이전 판단과 실제 가격 흐름을 비교한다.

## 데이터 원칙

- 실제 API 데이터를 우선한다.
- API가 없거나 실패하면 fallback을 쓰되 사용자 화면에 `(목데이터)`를 표시한다.
- 스냅샷, 설정, 리포트 발송 같은 서버 관리 값은 서버 저장소를 사용한다.
