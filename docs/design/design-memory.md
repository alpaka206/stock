# 디자인 메모리

이 문서는 Codex가 매번 사용자에게 디자인 방향을 다시 묻지 않도록 현재 저장소의 레이아웃, 분위기, 우선순위를 기록한다. 분위기나 방향이 바뀌면 이 문서를 덮어쓰지 말고 아래 변경 기록에 추가한다.

## 현재 기준
- 제품 성격: 금융 리서치 워크스페이스
- 기본 모드: 다크모드 우선
- 시각 밀도: 높음
- 구조: 카드 + 패널 + 그리드
- 목표: 뉴스처럼 가벼운 소비형 UI가 아니라 근거를 추적하며 판단하는 작업형 UI
- 사용자 언어: 한국어 우선

## 시각 장치
- 숫자와 퍼센트는 작은 카드 안에서도 바로 비교 가능해야 한다.
- 색상만으로 상승과 하락을 구분하지 않는다.
- 큰 차트와 dense grid가 같이 있어도 읽기 순서가 유지돼야 한다.
- 뉴스, 차트, 점수, 수급, 섹터가 끊기지 않는 흐름이 중요하다.
- 페이지는 4개로 고정하고, 세부 기능은 페이지 내부 패널과 탭으로 수용한다.

## 페이지별 레이아웃 기억
### 1) overview
- 상단 지수 스트립이 매우 중요하다.
- AI 시황 요약 카드는 짧고 단단하게, 오늘의 관전 포인트가 바로 보여야 한다.
- 히트맵은 섹터 강도를 빠르게 읽는 시각 장치다.
- 뉴스, 섹터 강도, 리스크가 바로 다음 화면 이동으로 이어져야 한다.

### 2) radar
- 이 페이지는 watchlist와 sector intelligence를 동시에 수행하는 핵심 작업 화면이다.
- 좌측 탐색, 중앙 grid, 우측 분석 패널 구조를 유지한다.
- grid는 단순 표가 아니라 오래 머무르는 작업 영역이다.

### 3) stock detail
- 중앙 차트의 비중이 커야 한다.
- 우측은 사용자 규칙과 프리셋 조건 설정 영역이다.
- 하단으로 갈수록 세부 정보가 많아지므로 정보 분류가 명확해야 한다.

### 4) history
- 과거 사건을 다시 읽는 화면이다.
- 차트와 이벤트 타임라인의 싱크가 중요하다.
- "왜 그날 움직였는가"를 서술형으로 설명해야 한다.

## 컴포넌트 스타일 기억
- Card 헤더는 짧고 직관적으로
- 설명 텍스트는 짧은 bullet보다 짧은 문단이 더 낫다
- 대형 그리드의 상태 저장, 컬럼 숨김, 그룹 상태 복원도 고려한다
- 금융 차트와 오버레이의 역할 분리를 명확히 한다

## 변경 기록
### 2026-03-15
- 최초 디자인 메모리 생성
- 사용자와 합의한 4개 화면 구조를 기준으로 삼기 시작

### 2026-03-16
- 공통 shell과 좌측 고정 navigation + 상단 context bar 구조를 유지한다.
- 카드 간격은 `--space-*`, 카드 padding은 `--card-padding` 토큰을 기준으로 맞춘다.
- 숫자와 퍼센트는 `JetBrains Mono` + `tabular-nums`로 고정해 비교를 쉽게 한다.
- 다크모드가 기본이지만 라이트모드도 동일한 정보 밀도와 대비를 유지한다.
- AG Grid는 한국어 locale과 dense research theme를 기본값으로 둔다.

### 2026-03-18
- overview는 "상단 지수 스트립 -> AI 시황 요약 -> 히트맵 -> 뉴스/섹터 -> 리스크 배너" 순서로 10초 읽기 흐름을 고정한다.
- overview 카드들은 단순 정보 제시가 아니라 `/radar`, `/stocks/[symbol]`, `/history`로 이어지는 탐색 진입점 역할을 해야 한다.
