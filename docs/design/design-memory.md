# 디자인 메모리

이 문서는 Codex가 **매번 사용자에게 디자인을 다시 묻지 않도록** 남기는 디자인 기억 저장소다.
레이아웃, 분위기, 우선순위가 바뀌면 이 문서를 수정하지 말고 **아래에 추가 기록**을 남긴다.

## 현재 기준선
- 제품 성격: 금융 리서치 워크스페이스
- 기본 모드: 다크모드 우선
- 시각 밀도: 높음
- 구조: 카드 + 패널 + 그리드
- 느낌: 토스처럼 가벼운 소비형 UI가 아니라, 근거를 누적해서 판단하는 전문가형 UI
- 사용 언어: 한국어 우선

## 시각 원칙
- 숫자와 퍼센트는 작은 카드 안에서도 비교 가능해야 한다.
- 색만으로 상승/하락을 구분하지 않는다.
- 큰 차트와 dense grid가 함께 살아야 한다.
- 뉴스, 차트, 점수, 수급, 섹터가 끊기지 않는 흐름이 중요하다.
- 페이지는 4개로 고정하고, 서브 기능은 페이지 내부 탭/패널/드로어로 수용한다.

## 페이지별 레이아웃 기억
### 1) overview
- 상단 지수 스트립이 매우 중요하다.
- 히트맵은 시황을 빠르게 읽는 시각 장치다.
- AI 시황 요약은 길지 않게, 핵심만 보여준다.

### 2) radar
- 이 페이지는 watchlist와 sector intelligence를 동시에 수행하는 핵심 작업 화면이다.
- 좌측 탐색 / 중앙 grid / 우측 분석 패널 구조를 유지한다.
- grid는 단순 표가 아니라 사용자가 오래 머무는 작업 영역이다.

### 3) stock detail
- 중앙 차트를 가장 크게 둔다.
- 우측은 사용자 규칙/프리셋/조건 설정 영역이다.
- 하단 탭은 정보량이 많아도 된다. 대신 분류가 선명해야 한다.

### 4) history
- 과거 사건을 다시 읽는 화면이다.
- 차트와 이벤트 타임라인의 싱크가 핵심이다.
- "왜 그날 움직였는가"를 한눈에 설명해야 한다.

## 컴포넌트 스타일 원칙
- Card 헤더는 짧고 직관적으로
- 설명 텍스트는 짧은 bullet보다 짧은 문단이 낫다
- 표/그리드의 상태 저장, 컬럼 토글, 그룹 상태 복원을 고려한다
- 금융 차트 위 오버레이(지지/저항/채널/이벤트 마커)는 레이어 분리를 명확히 한다
