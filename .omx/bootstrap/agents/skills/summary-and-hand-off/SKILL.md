# summary-and-hand-off

name: summary-and-hand-off
description: record compact evidence, risks, and next steps at the end of an iteration or session.

## Trigger
- at the end of an iteration
- at the end of a session
- when work becomes blocked and must be resumed later

## Do Not Trigger
- in the middle of active implementation unless a checkpoint is needed

## Rules
- lead with evidence and remaining gaps, not fluff
- mention verification commands and results explicitly
- keep the summary aligned with backlog and next-prompt files
- never include secret values
- make the next action obvious for the following iteration
